Technical & Concept Explainer

Cognify

A cognitive analysis interface that tracks the quality of how you think — not just what you read.

Version 0.1 — Hackathon Build April 2025 React · Tailwind · Anthropic-ready

1. What Is Cognify?

Cognify is a single-page web application that analyses conversation transcripts through the lens of cognitive quality. It does not fact-check. It does not summarise. Instead, it asks: How well did you think during this conversation?

The application extracts structured signals from any conversation — typically a human interacting with an AI assistant — and returns scores, flagged claims, engagement patterns, and a cumulative profile that evolves across multiple sessions.

Core distinction: Every other AI-adjacent tool focuses on the output — what was said, what was generated, what is true. Cognify focuses on the process — how critically, deeply, and carefully a person engaged with the information they encountered.

2. The Problem It Solves

2.1 The Epistemic Blind Spot

Humans have consistent patterns in where their critical faculties are strong — and where they are weak. A person may be rigorously skeptical about political claims while accepting health information uncritically. A student may ask excellent questions about engineering concepts but absorb nutrition myths without challenge.

These patterns are almost entirely invisible. No existing tool reveals them. We leave conversations with no feedback on how we engaged — only with whatever content we happened to receive.

2.2 The AI Amplification Problem

AI assistants have become a primary information interface for hundreds of millions of people. They are authoritative in tone, fluent in language, and capable of presenting false information convincingly. When a person with a blind spot in health reasoning encounters a confident AI-generated health claim, the result is uncritical acceptance at scale.

This is not a failure of AI. It is a failure of the epistemic feedback loop. Cognify closes that loop.

3. How It Works

3.1 Input

The user pastes any conversation transcript into the analysis field. The format is flexible — the extraction engine is designed to handle natural conversation formats (User: ... / Assistant: ...) as well as freeform discussion. A built-in demo injects a realistic pre-written conversation covering AI productivity, health claims, and skepticism of conspiracy content.

3.2 Extraction Engine

The application sends the conversation to an analysis function using the following extraction prompt (stored as a constant for full transparency and auditability): 

// EXTRACTION_PROMPT — the instruction sent to the analysis engine
const EXTRACTION_PROMPT = `
You are a cognitive analysis engine. Given a conversation transcript, extract:
1. Critical thinking score (0-100)
2. Depth score (0-100)
3. Engagement score (0-100)
4. Flagged claims with risk level (high/medium/low) and category tag
5. Topics discussed
6. Engagement signals (behavioural patterns)
7. Summary: 2-3 sentence narrative of cognitive patterns

Return strict JSON:
{
  "scores": { "criticalThinking": number, "depth": number, "engagement": number },
  "claims": [{ "text": string, "risk": "high"|"medium"|"low", "tag": string }],
  "topics": string[],
  "engagementSignals": string[],
  "summary": string
}
`;

In the current hackathon build, a mock adapter returns realistic hardcoded output matching this schema exactly. The adapter is isolated in a single function (runMockAnalysis) so that switching to a live Anthropic API call requires changing only that function — the rest of the application is unaffected.

3.3 Cognitive Scores

CT
Critical Thinking
DPT
Depth
ENG
Engagement
  • Critical Thinking (0–100): How much the user challenges, questions, and evaluates claims rather than accepting them passively.
  • Depth (0–100): How substantively the user explores topics — follow-up questions, nuance-seeking, mechanism-probing.
  • Engagement (0–100): How actively the user participates — response quality, contribution volume, topic coverage.

3.4 Claim Risk Mapping

Each significant claim in the conversation is extracted and classified:

  • HIGH RISK — Unverified, medically dangerous, or factually contested claims (e.g. "supplements can cure depression")
  • MEDIUM — Oversimplified, partially true, or context-dependent claims
  • VERIFIED — Well-supported, scientifically grounded, or appropriately qualified positions

3.5 Longitudinal Profile

Every analysed session is persisted to storage (using window.storage → localStorage → in-memory fallback). Across sessions, the application builds:

  • Topic frequency map: A visual tag cloud of subjects discussed, sized by recurrence.
  • Score trend line: SVG line chart of Critical Thinking and Depth across sessions over time.
  • Blind spots: Topic categories where HIGH RISK claims were repeatedly accepted.
  • Strengths: Topic categories where the user consistently demonstrates calibrated skepticism.

3.6 The Cognitive Fingerprint

The fingerprint is a dynamically generated second-person narrative paragraph synthesising all profile data into a plain-English description of the user's epistemic habits. Example:

"You engage most deeply with AI, productivity, health topics, where your critical thinking is most active and your questioning most structured. Across 4 sessions, your critical thinking registers as moderate (avg 55/100) with a depth score of 61/100 — indicating substantive exploration of ideas. You show a recurring pattern of reduced scrutiny around "health" and "nutrition" claims — these are areas where epistemic vigilance may be worth consciously raising. Your skepticism is well-calibrated toward science content. Your critical thinking trend is improving over time — you are asking better, more precise questions with each session."

4. Societal Relevance

4.1 The Misinformation Crisis

The World Health Organisation declared an "infodemic" alongside the COVID-19 pandemic — a recognition that misinformation spreads faster than corrections and causes direct harm. Health misinformation alone contributes to vaccine hesitancy, dangerous self-treatment, and preventable deaths. The root mechanism is not the existence of false information — false information has always existed — but the failure of individuals to apply consistent skepticism when encountering it.

Cognify addresses this at the source: not by removing false information, but by making visible the cognitive patterns that allow it to take hold.

4.2 AI and Epistemic Dependency

Studies on AI-assisted work show that users frequently accept AI output without verification, particularly in domains where they feel less confident. This creates a feedback loop: low confidence in a domain → high AI reliance → uncritical acceptance → no skill development → lower confidence. Cognify can break this loop by surfacing where reliance is highest and where scrutiny is lowest.

4.3 Education and Measurable Epistemic Growth

Critical thinking is widely cited as a core educational objective — and almost universally unmeasured. Standardised tests measure content recall. Essays measure communication. Nothing measures the quality of reasoning in context. Cognify makes critical thinking a quantifiable, trackable metric, enabling educators to demonstrate improvement, identify struggling students, and design targeted interventions.

4.4 Democratic and Civic Implications

Healthy democracies depend on citizens who can evaluate competing claims. The decline of epistemic norms — the erosion of shared standards for what counts as evidence — is widely documented as a threat to democratic function. A tool that gives individuals a personal accountability metric for their reasoning quality is, at scale, a civic infrastructure tool.

The core argument: You cannot improve what you cannot measure. Cognify makes epistemic quality measurable for the first time in a natural, conversational context.

5.3 Key Abstractions in App.jsx

  • EXTRACTION_PROMPT — the analysis instruction, stored as a constant for parity between mock and real modes
  • runMockAnalysis() — isolated adapter; replace with Anthropic API call to go live
  • storageGet() / storageSet() — storage wrapper with three-tier fallback
  • aggregateProfile(sessions) — pure function: sessions array → profile object
  • generateFingerprint(profile) — pure function: profile → narrative string
  • CircularScore, ClaimBadge, TrendChart, TopicMap — presentational components

5.4 Enabling Real API Integration

To replace mock analysis with a live Anthropic call:

  1. Add VITE_ANTHROPIC_API_KEY=your-key to a .env file
  2. Replace runMockAnalysis with a fetch call to the Anthropic Messages API using EXTRACTION_PROMPT + the conversation as user content
  3. Parse the JSON from the model's response — schema is identical to the mock output

No other code changes are required.

  9. Reload page — data persists via localStorage

7. Limitations and Honest Assessment

This is a hackathon prototype. The analysis in the current build is mock/heuristic. Scores are derived from simple signals (question count, word count) rather than real cognitive analysis. The value of the architecture is in the structure, the data model, and the UX — not the current precision of the mock engine.

  • Mock only: Real cognitive scoring requires live LLM analysis. The prompt and schema are production-ready; the adapter is not.
  • Client-side API keys: Direct browser-to-Anthropic calls expose API keys. A backend proxy is needed for production.
  • No authentication: All data is stored locally; no cross-device sync.
  • Single-conversation granularity: The profile improves with more sessions but requires user effort to supply transcripts manually.

8. Vision

The long-term vision for Cognify is a standard layer for epistemic quality monitoring — embedded in AI assistants, browser extensions, and educational platforms. In the same way fitness trackers normalised tracking physical health metrics, Cognify aims to normalise tracking cognitive health metrics.

A future version analyses conversations in real time, aggregates anonymised patterns at population scale, and provides researchers, educators, and policymakers with the first empirical dataset on how critical thinking varies across domains, demographics, and AI usage patterns.

The core thesis is simple: in a world where AI generates the answers, the quality of the questions you ask — and the rigour with which you evaluate the responses — is the only meaningful measure of cognitive competence left. Cognify makes that quality visible.
